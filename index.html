Save the Earth

Interactive Game Application

Description of Functionality: Save the Earth teaches the user about the benefits of certain energy resources over others. This is accomplished by interacting with the user directly and having them experiment through trial and error. The user has income and they have to replace coal energy sources with other sources that have different costs and benefits. If the user reaches the energy and pollution goal by the end of the game they win otherwise they lose and start again and try building different energy sources. As the user play enough times they will learn that certain resources are cheaper with greater benefits than others.

Description of Purpose: This program was created for a school project. The requirements were to create an application that focused around one of the yearly goals put out by the UN. Specifically the goal that revolved around converting energy sources to clean sustainable sources.

Lines of Code: 1617

Technical Skills: This product used SFML a GUI I have never worked with before. Textures were loaded and saved into data structures and all buttons were custom made. There are mouse events for clicking on power plants and buttons including several overlays that are triggered based on game variables.

Role: Since this was a group project my main role was creating the game structure and making sure all data structures load and save their data before the program runs. I made some of the buttons that transition between screens and made functions to check for all the user inputs.

Looking Back: I learned more about data encapsulation, checking for events, and how sprites and textures work.

Drawing Frame

A simple drawing application for the desktop.

Description of Functionality: Drawing Frame is a program application that allows users to select a shape and draw it to the screen. The user is able to manipulate the line width, outline color, and fill color. The drawing window may have multiple shapes and colors printed to the screen including an option to clear the canvas. Other features that are included allow the user to resize, minimize and close the window through mouse events. Additionally, the program notifies the user of what current shape they have selected through button down display info on the screen.

Description of Purpose: This project was modified from an existing drawing application. The project was created to familiarize myself with the wxWidgets GUI and functions that are typically used for creating input events. Including, learning about fractals and how they can be implemented recursively.

Lines of Code: 275

Technical Skills: The program draws shapes through preexisting shape function that are in the wxWidgets library. The fractal is created through a self-made recursive function.

Role: I worked on the main draw files from the preexisting application which included, creating draw functions or implementing them while updating the global variable that kept track of the current shape status.

Looking Back: I learned how to draw using a GUI library and how to use recursion to solve more complex problems. Looking back I would like to clean up the recursive function used to create the fractal.

The Maze Adventure

A simple maze game.

Description of Functionality: The maze adventure has a user go through a set of mazes. Each maze has a start point and end point. The user will have to collect keys to corresponding doors to unlock them and progress through the level. The user navigates through each level as fast as they can to get a new high score. The user can view their score on the main screen and determine if they want to play again to beat it.

Description of Purpose: This program was created for a school group project. The requirements were broad and we just had to create something that incorporated the things we learned that year. My motivation for this specific project was to get familiar with user input and game character interaction.

Lines of Code: 1286

Technical Skills: The project loaded and saved data from a file. Two dimensional arrays were used to display the layout of the screen and character movement. After the user moved the character, the screen has to refresh and show the current player location.

Role: I created the frame work of the program. Framework consisted of moving the player, updating files outside the program, getting events from the user, and updating player movement.

Looking Back: I learned a lot about how to move a character across a two dimensional array. In addition to moving the character in code, I also learned how to give the illusion of player movement and how to refresh the screen. The screen refresh could have been implemented better.
